適切なツールを選定・導入する
開発者との連携強化（日礼で確認したいことを確認しやすくする環境構築など）
議論に参加することで、QAが変更内容を深く理解し、並行してテスト計画とテストケースの作成を正確に行えるようにする
常に開発側の負担軽減を意識する
開発チームにQA観点を浸透させる

手順
1.組織図
2.QAマネージャー
3.全体スケジュール作成と共有 
4.責任分界（非機能テストの責任まで負うのか等）
5.上層部へグループ方針を伝える 
6.リリースフローやテストフロー図、不具合レベル指標（責任範囲や、チームの作業タスクを決めるため） 
7.ドキュメント用意（テスト計画書、テスト設計書、テスト仕様書、不具合起票方法）
(8).新規参画メンバーのタスク（未経験者が参画した場合のナレッジ共有手法の確立） 
9.要件定義レビューにQA参加する形式の考案
10.作成したフローや指標を元に、稼働してみる 
11.課題点は都度チーム内で共有し、フローや指標を見直す（これを繰り返しながらチーム運営する）
